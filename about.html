Om

Den här uppgiften är sammankopplad med det tidigare delmomentet i kursen och använder sig av SQLite för databasanslutning. Webbplatsen använder sig av CRUD-operationer för att hämta, ladda och radera information lagrad i databasen. Genom att använda fetch API och metoderna GET, POST och DELETE kan datan lagrad i databasen ändras, utan att sidan använder sig av Express, EJS eller andra program. Detta är en fristående Frontend-sida som endast består av HTML, CSS och Javascript. För att automatisera arbetsprocessen har Parcel använts. Koden innehåller en arbetskod för PUT-operationen. Jag fick det inte att fungera helt men lät den ändå vara kvar för referens i framtida projekt. Slutsatserna jag dragit är att jag behöver ägna ganska mycket tid åt backend för att förstå delarna, men nu i slutet av detta moment förstår jag äntligen syftet med uppgiften och kan se fördelarna med att använda fetch API och metoderna och tillvägagångssättet som används i detta moment.
